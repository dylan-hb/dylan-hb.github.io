分类模型评估中的名词解释

发表于 2018-05-17 | 分类于 分类性能度量指标

一、基本定义

1、混淆矩阵

混淆矩阵可以对预测值与真实值做可视化展示。混淆矩阵用n行n列的矩阵表示，矩阵的列表
示的是模型的预测情况，矩阵的行表示的样本的真实情况。如下图所示：

TP: True Positive （真正），预测为正，实际为正（预测对了）。
FP: False Positive （假正），预测为正，实际为负（预测错了）。
FN: False Negative （假负），预测为负，实际为正（预测错了）。
TN: True Negative （真负），预测为负，实际为负（预测对了）。

2、准确率(Accuracy)

准确率表示的是被预测对的数据（TP+TN）占所有数据(TP+FP+TN+FN)的比例。Accuracy=（TP+TN）/(TP+FP+TN+FN)。反应的是分类器对整个样本的判定能力。

3、精确率(Precision)

精确率表示的是被预测为正（TP+FP）的数据中，真正为正（TP）的数据比例。
Precision=TP/(TP+FP)。精准率反应的是对预测数据的真确率。

4、召回率(Recall)

召回率表示的是预测为正，且真为正的数据（TP），占所有为正的数据的比例（TP+FN）Recall=TP/(TP+FN)

5、F1值（F1）

为了提升召回率，就需要尽量更多的预测数据为正，带来的后果就是精准率的下降（预测的越多，理论上错的概率越大）。F
-Score是为了兼顾分类模型的精准率、召回率，可以看作是模型精准率和召回率的一种加权调和
平均。
F-Score= F-Score = (a2+1) * Precision * Recall / a2 *(Precision +Recall) 当a
=1时，F-Score 即为 $$ F1 = 2 *（Precision * Recall）/ (Precision +Recall
) $$，最大值是1，最小值是0。

6、TPR

TPR表示的是真正例率，即：预测为正例，且真正为正例的数据（TP），占所有正例的比例。TPR = TP/(TP+FN) (同召回率)

7、FPR

FPR表示的是假正例率，预测为正，但是实际是负的数据（假正）占所有负样本（TN+FP）的比例。FPR = FP/(TN+FP)

8、ROC曲线

ROC是以TPR为Y轴，FPR为X轴，从坐标（0，0）到坐标（1，1）的曲线
(也可能不是从0，0坐标开始) 如下图1所示：

上图解释：
对于一个分类器，预测某一个记录为正的概率为p，把概率p从高到低排序后。依次取概率作为阈值，把预测样本中概率值大于等于阈值的数据认为是正样本，小于阈值的数据认为是负样本。这样就能得到一系列的TPR，FPR坐标点。多组TPR、FPR坐标点组成了上述ROC曲线。
举例：
下表中总共20条样本记录，其中正样本P为10个，负样本N为10个。
当阈值为0.9的时候，第一条记录为预测为正样本，此时第1条记录实际上是真正样本，所以TP=1，TRP = TP/P = 1/10 =0.1。FP = 0，FPR = FP/N = 0/10 = 0。所以当阈值为0.9的时候，得到坐标点（0,0.1）

当阈值为0.6的时候，第1、2、3、4样本被预测为正样本，此时1、3、4也是真正样本，第3条记录是负样本，所以TP=3，TRP = TP/P =3/10 = 0.3。FP = 1，FPR = FP/N= 1/10 = 0.1。所以当阈值为0.6的时候，得到坐标点（0.1,0.3）

最终把20组的FPR、TPR坐标点绘制到x、y的平面图上得到ROC曲线图。如下图2所示。

上文提到，ROC的曲线也可能不是从0，0坐标开始的，如上图就是从（0，0.1开始的）。只有当阈值=1的时候，此时所有预测数据都为负，没有正例，也就不存在正正例率，假正利率，所以此时TPR=0，RPR=0
由于真正率越高，分类效果越好，所以一个好的分类器的ROC曲线应该是从（0,0）出发，到（0，1）再到（1，1）的曲线。也就是曲线离Y轴越近越好。
